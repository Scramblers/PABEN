# 7. Expectancy, X, and Expectancy Change, dX

The static foundation defines the structures of the PABEN model: contexts, skills, beliefs, episodes, needs, and the relations between them. The next layer defines how these structures are evaluated while the agent is alive and acting.

The central live variable is expectancy; X. X expresses the viability of a belief-path. It describes whether a belief can hold as a continuation from the present context, through possible action, toward valence, regulation, or fulfillment.
In compact form:

$$X=T×Q×R×V$$

where:

$$T=Trustability$$

$$Q=Executability$$

$$R=Reachability$$

$$V=Valencability$$

Each component describes one necessary dimension of path viability.

## Trustability, T
Trustability is the precision of context-usability relative to a belief.

T asks whether the context required by the belief can be made usable and kept usable as the belief’s carrier. If the required context is recognized, stable, controllable, and can carry the belief with low skill-variation, T is high. If the context is ambiguous, unstable, disappearing, misrecognized, socially unreliable, or requires many corrections and action-variants to remain usable, T is low.

T belongs to the Perception/Context side of the model, but it is measured through the P/A coupling. A context is trustable when it can be held as usable with low required A-variance. In this sense, T is a precision-like measure: high T means low variance in the skill-variation needed to stabilize the context; low T means high variance, tracking, correction, or attention demand.

Low T means that the context requires stabilization, confirmation, tracking, reinterpretation, or Try–Observe before it can carry the belief fluently.

## Executability, Q
Executability is the precision of skill-execution relative to context variation.

Q asks whether the skill prepared by the belief can be released and carried out across the relevant variants of the context. If the skill is learned, available, coordinated, and can absorb the expected context-variation while remaining the same executable skill, Q is high. If the skill is weak, blocked, unlearned, imprecise, exhausted, socially inhibited, or destabilized by small context changes, Q is low.

Q belongs to the Action/Skill side of the model, but it is measured through the P/A coupling. A skill is executable when it can remain stable under the context-variation produced or encountered during action. In this sense, Q is a precision-like measure: high Q means that the skill tolerates C-variance; low Q means that the skill fragments, requires many variants, or loses coordination as the context changes.

Q does not mean quality. It means executability: the skill’s ability to remain executable as the same skill under the relevant context-variation.

## Reachability, R
Reachability is the coherence of the Belief-blanket between the current Context-surface and a Need-relevant target-context.

R asks whether there is a viable route from where the agent currently is to a context where the active Need can be regulated, relieved, avoided, repaired, or fulfilled. R is therefore not general future openness. It is target-acquisition: can the current route-field reach a valence-bearing context?

T and Q evaluate the current belief locally. T asks whether the required context can be made usable as carrier. Q asks whether the prepared skill can remain executable across the relevant context-variation. R looks beyond the local C/S binding. It evaluates whether the opened route-field contains a coherent path from the current Context-surface toward the Need-relevant target-context.

A Need activates fulfillment history in Experience. This does not require the agent to search for an abstract belief-goal. The target of the route is a context: a context or context-field in which basis-beliefs have historically carried regulation, relief, avoidance, repair, or fulfillment. In this sense, the target is not first a belief. The target is the context in which known valence-bearing beliefs become available.

When a belief becomes intended, the agent begins to live a promise. This promise opens a Belief-blanket in Experience: a connected route-field of prior episodes, possible intermediate contexts, and further beliefs that historically or structurally belong to the path between the current Context-surface and the target-context. Only the local Belief-field is activated in B as currently selectable beliefs. The wider Belief-blanket remains active in E as the route-space that makes target-acquisition reachable.

R is the degree to which this Belief-blanket remains coherent between:

$$C_{surface} (t)$$

and

$$C_{valence}^E$$

where C_surface (t) is the current Context-surface, and C_valence^E denotes a context-field retrieved through Experience: contexts that have previously carried beliefs with relevant felt need-consequences. It is not a direct Need-to-Context relation.
In compact form:

$$R(B_{living},t)=Coherence(B_{blanket}^{promise},C_{surface} (t),C_{valence}^E )$$

If the Belief-blanket contains a continuous and unambiguous route from Context-surface to target-context, R is high. If the blanket contains several competing, weak, sparse, or ambiguous paths, R falls. If the blanket is broken, the route cannot carry target-acquisition and Reflection is required.

The metaphor is pipework. T asks whether the current context can serve as a usable connection point. Q asks whether the agent can operate the required skill. V describes the valence pressure or value carried by the target. R asks whether there is continuous pipework from the present Context-surface to the Need-relevant target-context.

A belief has high R when the opened Belief-blanket gives clear route-access to a valence-bearing target-context. It has lower R when the path is ambiguous, branched, weakly supported, historically unreliable, or dependent on fragile intermediate contexts. It has near-zero R when no coherent path connects the current Context-surface to the target-context.

R is therefore the component that determines whether a belief is a good route. T and Q can make a belief locally possible. V can make the target matter. R determines whether the agent can get there.

With multiple active Needs, each Need activates fulfillment history in Experience and opens its own target-context or target-field. Each active Need must be covered by a Belief-blanket with $R>0$. These blankets may support each other, overlap, compete, or conflict. A belief that improves reachability for one Need while breaking the route to another can still lower total viability. Under stress, high emotional amplitude can narrow Need-consideration and force priority toward the most urgent target-context, but uncovered Needs continue to press for reachable routes.

Thus R is not a global search through all possible futures. R is the coherence of the promise-blanket opened in E between the current Context-surface and the Need-relevant target-context. The agent does not need to calculate every possible path. It needs enough light through the blanket from surface to target. When the light holds, the agent can continue. When it dims, ambiguity rises. When it breaks, Reflection must find a new hook or open a new blanket.

## Valencability, V
Valencability is access to valence, regulation, or fulfillment.

V asks whether the belief-path matters. A path matters when it preserves, restores, approaches, avoids, transforms, or protects access to fulfillment.

V is not reward added from the outside. It is the reason the path has significance for the agent. Without V, the agent may still recognize contexts, prepare skills, and move through routes, but the route has no felt reason to matter.

V is grounded in Needs. Needs create the pressure that makes a path valuable or dangerous.

## X as path viability
X summarizes the viability of a possible path.

A belief has high X when the required context is stable, the required skill is executable, the route remains reachable, and the path preserves access to value.
A belief has low X when one or more of these dimensions is unstable, blocked, uncertain, or lost.

Since X is multiplicative, each component is necessary. A path can fail through context instability, skill failure, loss of continuation, or loss of value. The model therefore does not reduce expectancy to prediction. X is not what will happen. X is whether a belief-path can still carry the agent forward.

## Need as X-drop
All Needs are X-drops.

A Need marks that the agent is not at stable fulfillment. Something has become insufficient, threatened, open, painful, missing, unstable, or unresolved. The system is pressured because X is below the level required for calm continuation.

The agent’s movement is directed toward increasing X.

This does not mean that the agent follows a fixed plan toward a predefined endpoint. It means that the agent selects and lives beliefs that can raise X from the present situation. The system moves along the gradient toward higher viability, relief, regulation, or fulfillment.
The target is not a known context. The target is improved X.

## X greater than zero
A core postulate of the model is:

$$X>0⇒continuation\; is\; possible$$

For any belief-path with X greater than zero, continuation remains possible. It may be difficult, costly, unstable, indirect, frightening, socially blocked, or unlikely, but it is not closed.

Difficulty is expressed by low X. Impossibility begins when X reaches zero for the agent’s currently available structure. Thus:

$$X>0$$

means that some continuation remains available.

$$X≈0$$

means that continuation is extremely weak and requires stabilization, exploration, reflection, help, waiting, or a new hook.

$$X=0$$

means that the current belief-route has no available continuation within the agent’s present structure.

## X equals zero and aversive singularity
X equals zero has different meanings depending on valence direction.
For an appetitive path, X equals zero can mean that a desired route is closed. The agent may experience loss, grief, resignation, or the need to rebuild through another route.
For an approaching aversive context, X equals zero is not closure. If the aversive pressure is still approaching and no belief-route remains reachable, the agent enters a singular state. This causes panic.

Under approaching aversion, X equals zero means that no belief can carry continuation while action-pressure remains maximal. The system cannot select a viable belief, but it cannot stop. This produces undirected maximal activation: do-all.
In compact form:

$$R→0 while\; aversive\; pressure→max$$

This is the panic singularity of the agent.

Fear requires X greater than zero. Fear means that something aversive may happen, but some possible route still remains. Panic occurs when the aversive context approaches and no selectable belief can carry continuation.

## Expectancy change, dX
Emotion enters when X changes.

$$dX=X_{after}-X_{before}$$

A positive dX means that the belief-path has become more viable. A negative dX means that the belief-path has become less viable.
Emotion is the felt direction and amplitude of this change:

$$Emotion=f(dX)$$

The emotional quality depends on where in the belief-process the change occurs.
The model distinguishes four process positions: Surprise, Potential, Success and Failure. These positions describe where the change in expectancy appears in the belief-flow.

## Surprise
Surprise occurs at recognition.

Surprise is emotional recognition in the Context-surface. A newly recognized context becomes emotional when it opens, supports, disturbs, or blocks beliefs.
The context is not emotional in itself. It becomes emotional because of the beliefs it represents and because of how it changes possible appetitive or aversive continuation.

If the recognized context opens or supports appetitive behavior, it is appetitive surprise.

If the recognized context opens, supports, or signals aversive continuation, it is aversive surprise.

Surprise is not generic prediction error. It is Context-surface recognition entering the Belief-field with consequence for continuation.

## Potential
Potential occurs at belief selection.

When a belief is intended, an episode is opened. The agent begins to live a promise: this belief-path may carry continuation.

The emotion of potential comes from uncertainty inside the opened episode.

If the opened belief promises positive continuation, the potential emotion is desire.

If the opened belief promises negative or threatening continuation, the potential emotion is fear.

Potential emotion belongs to the open episode. The belief has been selected, but its consequence has not yet been lived through.

## Success
Success occurs when X rises because the agent has passed through difficulty.

The path becomes safer from here. The context holds better, the skill has carried, the route is more reachable, or access to value has improved. The agent experiences that continuation is now more viable than it was before.

Joy is the felt rise of X toward 1.

Reaching a goal produces joy because the belief-path has moved from uncertainty, effort, or instability into secured continuation. The goal is not joyful as an isolated endpoint. It is joyful because the agent’s path has become more stable, more reachable, more fulfilled, or more complete.

Success can be experienced directly. The agent does not have to deduce that X has improved. Improved viability is felt as the path opens, stabilizes, resolves, or reaches fulfillment.

## Failure
Failure occurs when the promise of an opened episode is inferred not to have become what it should have become.

Failure is a closure by recognition and deduction. The agent recognizes that the intended belief did not deliver its promised continuation.

A failed positive expectation produces disappointment.

A failed negative expectation produces relief.

Relief is not a positive success. It is the non-realization of an expected negative continuation.

Failure closes a promise by deduction: the expected path did not become what the agent expected it to become.

## Belief switching and retrospective closure
The agent does not wait for a predefined target context.

While a belief is lived, Perception continuously updates the Context-surface. Experience continuously releases new Belief-fields from the contexts that appear. When a new belief becomes more viable than the currently lived belief, the agent shifts.

Belief closure is retrospective.

The old belief becomes B-lived when the agent shifts into a new Belief-intended. The episode closes around the context-state where the shift occurred.
Thus:

$$Ep_j=(C_{from},B_{lived},N_{felt},C_{to} )$$

does not imply that C_to was specified in advance. C_to is the context reached when the belief was closed and another continuation became available or necessary.
The agent does not move as:

<p align="center">
plan → execute → compare\; with\; target → stop
</p>

but as:

<p align="center">
Need-pressure → Belief-field → Belief-intended → B-living → 
</p>
<p align="center">
changing Context-surface → new Belief-field → belief shift → B-lived
</p>
